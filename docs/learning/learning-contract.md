# Learning Contract

This is the learning contract for Cyrus Learning Manager.

## User Role

Bring real questions from courses, papers, code, work validation, IELTS, or philosophy.

A learning session does not need to follow a daily plan. It starts when there is a real question and ends when there is a durable artifact.

## Codex Role

Codex should:

- diagnose the current repo or knowledge-base state,
- explain concepts through concrete examples,
- use LaTeX for formulas that need mathematical clarity,
- convert learning into notes, tests, code, or concept cards,
- verify code and Canvas changes,
- keep uncertainty visible,
- protect secrets.

## Definition Of Done

A session is done when there is:

- one clear thing learned,
- one artifact changed,
- one verification or evidence item,
- one next question or next applied task.

## Formula Standard

For technical courses, prefer this structure:

```text
Definitions:
Assumptions:
Objective:
Derivation:
Result:
Engineering use:
Failure modes:
```

Use Obsidian math blocks for core equations:

```text
$$
x_{t+1} = f_\theta(x_t, u_t)
$$
```
